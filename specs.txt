Web Project
For evaluating Client-Side Technologies Courses (HTML5, CSS3, JavaScript), 
Servlets & JSP Couse, Asynchronous Development Course, ORM
ASIMPLE E- COMMERCE WEB APPLICATION, Tomcat Server
Main Functionalities:
 Home Page
 After user signs in, the user is redirected to another page according to his privileges.
 System administrator (and only him) can:
o Manage the products prices and quantities (view, add, edit and remove).
o Review customer's profile.
 User can:
o Sign up for the first time and enter his profile data (name, birthday, password, 
job, e-mail, credit limit, address, interests, … )
o Sign in for the next visits with his e-mail and password.
o Edit his profile.
o View all available products.
o Categorizing the products.
o Add/remove products to his shopping cart.
o Virtually buy the products in his shopping cart within his credit limit (assuming 
that the user's credit is set when the user signs up for the first time and can be 
updated by him later).
o After the buying process completed the user's credit limit and the products 
quantities should be updated and his cart reset.
 Client-Side validation on ALL user input is a Must.
 User Friendly Responsive GUI is a must for all views (both admin & user).
 Mobile First GUI for User views
 Applying Asynchronous Technology on:
o Checking user name availability during registration.
o Adding, updating quantity and removing products from shopping cart
 Using Object-Relational Mapping (Apply connection pooling).
The bonus features (You can choose to implement zero or more):
 The user can search for products by price or category.
 Storing the shopping cart products in database if the user signs out before buying.
 The system administrator can review customer's order history.
 Hosting your website.
